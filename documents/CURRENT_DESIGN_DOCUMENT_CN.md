# VASP Show 当前设计文档

更新时间：2026-05-27

## 1. 产品目标

VASP Show 的目标是成为面向计算材料、聚合物和反应路径数据的高质量科研数据库界面。设计重点不是营销展示，而是让科研用户快速完成数据理解、筛选、结构检查、属性比较和溯源引用。

## 2. 设计原则

- 信息密度适中：避免大段纯文字，也避免装饰过多。
- 科研可信：每个数据集突出来源、计算方法、规模、单位和缺失字段。
- 结构优先：材料数据库首先要让用户看到结构、晶胞、元素组成和关键性质。
- 可比较：同一页面提供图表、筛选和表格，方便快速判断数据分布。
- 可扩展：不同数据集允许定制详情页模块。

## 3. 视觉风格

当前采用清爽科研配色：

- 主色：明亮但不过饱和的蓝色
- 辅助色：绿色、青色、紫色、黄色、红色均降低冲击感
- 背景：浅灰蓝白
- 卡片：白色或浅蓝灰
- 边框：低对比细边框
- 夜间模式：深蓝灰背景，保留较高可读性

配色参考方向：

- 生信平台常见的状态标签色
- Ant Design 的按钮与状态层次
- Materials Project 类材料数据库的信息组织方式

## 4. 首页设计

首页是整个数据库的第一入口，当前包含：

- 顶部导航：数据中心、数据发现、智能助手、模型、工作流与数据接入
- 主题切换：日间/夜间
- 计算数据 Hero：说明当前覆盖分子、聚合物、二维材料、晶体、MOF 和反应路径
- VASP/DFT 视觉示意：原子、轨道和晶格图形
- 数据域概览：分子构象、聚合物、二维/晶体、反应路径、MOF
- 元素图谱：高频元素柱状图
- 数据集卡片：类型标识、规模、原子数、方法、基组/赝势、元素芯片
- 数据发现入口：跳转至面向筛选任务的独立检索页面
- 全局底部：数据合作与接入、技术支持助手、登录与权限和联系信息承载位

## 5. 数据发现页设计

数据发现页承担“选择合适数据源”的任务，不将不断增长的数据集简单堆叠在首页。当前包含：

- 模糊检索：数据集名称、描述、DOI、元素、性质、软件、泛函和基组/赝势
- 分类筛选：小分子构象、聚合物、二维材料、晶体材料、MOF、反应路径
- 计算筛选：方法、交换-相关泛函、基组/赝势设置、计算软件
- 内容筛选：原子数区间、包含元素、多项必备性质字段
- 结果展示：展示记录数、结构规模、计算设置、展示策略、性质标签和元素标签
- 条件保持：筛选状态保存在 URL 中；进入数据集时继续传递原子数范围

性质名称遵循数据语义，例如使用广义“能量”作为发现入口，同时区分“总能量”“每原子能量”“原子化能”，避免将不同物理量混为一类。

## 6. 数据集记录页设计

记录页承担“快速理解一个数据集”的任务，当前包含：

- 数据集头部仪表盘
- 数据集类型标识
- 简介、规模、原子数范围和元素标签
- 论文 DOI、数据 DOI 或官方数据门户来源链接
- 材料性质图表：
  - 原子数分布
  - 能量分布
  - Band gap / HOMO-LUMO gap 分布
  - 元素组成统计
  - 字段可用性统计
- 搜索与筛选：
  - 关键词
  - 能量范围
  - 原子数范围
- 记录表格：
  - 名称
  - composition / SMILES
  - 原子数
  - 能量
  - gap
  - 电荷
- 交互说明：记录筛选为服务端筛选，通过“检索记录”按钮或回车提交，并展示匹配记录数；概览图表保留为全数据集基准分布，不随当前筛选重算。
- 数据集下载入口始终可见；游客和注册用户点击批量导出时会收到准确的角色权限说明。

## 7. 详情页设计

详情页承担“检查单条结构和属性”的任务，当前包含：

- 面包屑导航
- 记录标题、数据集来源、DOI 和软件/平台信息
- 关键指标卡片：
  - 原子数
  - 元素种类
  - 总能量
  - gap
  - 电荷/自旋
- 3D 结构查看：
  - 球棍结构
  - 键连线开关
  - 周期晶胞开关
  - 有 lattice 时显示真实晶胞
  - 无 lattice 时显示坐标包围盒
- 元素组成条形图
- 属性标签页：
  - 结构坐标
  - 计算参数
  - 热力学
  - 力学性能
  - 电子结构
  - 元数据
- 字段呈现原则：只有源数据确实存在的属性才显示；空标签页自动隐藏，避免专家把“暂无数据”误认为有效结果。
- 单条记录提供 JSON 下载入口，包含当前展示中真实存在的属性与可用坐标；游客点击时提示注册登录要求。

## 8. 账号与权限设计

账号入口位于顶部用户区域，支持登录和注册。登录后直接返回数据中心；超级管理员在顶部看到独立的“权限管理”入口，用户列表中超级管理员固定置顶。角色控制数据获取能力：

| 角色 | 界面能力 |
|---|---|
| 游客 | 浏览数据集、筛选记录、查看 3D 结构与公开来源链接 |
| 注册用户 | 游客能力，以及使用智能助手、下载单条记录 JSON、提交候选数据集 |
| 管理员 | 注册用户能力，以及下载数据集全部展示记录 |
| 超级管理员 | 管理员能力，以及管理用户角色、审核数据来源与推进接入管线 |

权限设计以“数据可发现、批量获取受控”为原则；来源论文和官方发布页对科研用户保持可见，应用内批量导出受账号控制。对于已经渲染给游客的单条 3D 结构，坐标数据无法在浏览器端做到绝对防复制。

## 9. 智能助手页设计

智能助手页服务于“带着数据提问”的研究流程，而不是开放式闲聊：

- 全局入口：基于全部数据集目录咨询适合的数据来源。
- 数据集入口：从记录页进入时携带当前数据集，询问性质、方法、规模与适用任务。
- 结构入口：从详情页进入时携带当前记录，询问已存在字段、能量与结构信息。
- 界面结构：左侧显示受控数据上下文与问题入口，右侧为对话区。
- 可信原则：回答下方展示所依据的数据集/记录来源；未提供的字段不作为已知事实陈述。
- 访问控制：游客可以查看数据，但登录用户才能发起推理请求，控制 GPU 资源使用。

## 10. 数据接入中心设计

数据接入中心承担“发现来源、邀请投稿、受控发布”的任务：

- 公开来源区：陈列人工维护的候选数据门户、API 或论文附属数据入口。
- 管线视图：来源提交、来源审核、格式适配、入库校验、发布更新五个阶段。
- 投稿表单：注册用户最少提供数据集说明和一个网页链接；名称、类型、格式、许可证和可用字段均为选填。
- 审核队列：超级管理员批准或退回来源，并将合格来源推进到格式适配阶段。
- 可信原则：异构外部数据不会因为提供了 URL 就直接写入展示库；需要解析适配器、字段/单位映射与抽样校验。

## 11. 当前可展示与不可展示

当前适合展示：

- 原子数分布
- 能量分布
- Band gap / HOMO-LUMO gap 分布
- 元素组成
- 3D 坐标
- 周期晶胞线框
- DOI、软件、泛函、基组/赝势
- OpenPoly、JARVIS、2DMatPedia、QMOF 等补充属性

暂不适合强行展示：

- 能带结构图：当前统一库没有完整 band path/eigenvalues
- DOS 图：当前统一库没有 DOS 曲线数据
- 反应路径能量曲线：Transition1x 当前只展示第 0 个代表图像，未懒加载整条路径
- VASP INCAR/KPOINTS/POTCAR 细节：源数据未统一结构化

## 12. 下一步设计建议

优先级从高到低：

1. 增加元素周期表热力图，用于首页和数据集页。
2. 增加 POSCAR/CIF/XYZ 等领域文件导出。
3. 给晶体材料详情页增加晶格参数 a/b/c/alpha/beta/gamma。
4. 给 JARVIS/QMOF/2DMatPedia 增加 band gap、formation energy、energy above hull 的专属图表。
5. 给 Transition1x 增加反应路径懒加载和能量曲线。
6. 给 ANI-1x 增加任意 conformer 懒加载。
7. 增加推荐引用格式和 BibTeX 导出面板。
8. 增加字段完整度面板，明确每个数据集有哪些字段缺失。
9. 增加 API 文档页面，面向外部用户和服务器部署使用。
10. 为智能助手增加论文原文检索、引用证据面板、反馈评价与管理端用量审计。
11. 为获批公开来源建设定时扫描、下载缓存、格式适配器和发布审计任务。

## 13. 世界一流数据库还需要的能力

- 数据版本管理和构建日志
- 数据质量校验报告
- 完整单位体系
- 结构相似性搜索
- composition/元素组合高级筛选
- 晶体结构的空间群、晶系和周期信息
- 电子结构曲线数据支持
- 批量下载和可复现实验包
- 稳定公开 API
- 每条记录可引用、可下载、可追踪来源
