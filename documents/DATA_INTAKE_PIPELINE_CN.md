# VASP Show 数据接入中心流程说明

本文档说明当前项目中“数据接入中心”的完整工作流。当前系统采用稳妥的半自动接入策略：系统负责公开来源发现、候选评分、链接与元数据预校对、样本解析和展示库写入；超级管理员负责关键审核、发布控制和撤回。

## 1. 总体流程

```text
公开 API 搜索
→ 候选数据集
→ 自动评分/校对
→ 超级管理员转入接入队列
→ 来源审核
→ 下载样本并格式预检
→ 字段映射或结构解析
→ 半自动写入展示库
→ 质量验证
→ 发布/隐藏/撤回
```

## 2. 公开 API 搜索

系统从公开数据源检索可能适合收录的计算数据集。当前已接入的数据源包括：

| 来源 | 接入方式 | 当前作用 |
|---|---|---|
| DataCite | DOI 元数据 API | 跨 Zenodo、Figshare、Materials Cloud 等仓库检索 DOI 元数据 |
| Zenodo | 直接 records API | 获取数据集记录、DOI、许可证、文件下载链接、大小与格式 |
| Figshare | 直接 articles API | 获取数据集详情、文件下载链接、标签、分类、许可证 |
| Materials Cloud | 经 DataCite 间接发现 | 通过 DOI 元数据发现 DFT 工作流、晶体结构、AiiDA 归档 |
| NOMAD | datasets API | 获取公开数据集名称、DOI 与归档入口 |
| Dryad | 直接 API | 获取跨学科开放数据仓库中的 DOI、许可证和数据包入口 |
| OpenAIRE | datasets API | 获取欧洲开放科学聚合平台中的数据集元数据 |

检索关键词来自后端配置，典型关键词包括：

- `VASP DFT dataset energy force materials`
- `density functional theory dataset polymer energy`
- `DFT structures forces HDF5 dataset`
- `materials phonon dielectric bandgap dataset`
- `computational materials dataset CIF POSCAR`

系统支持定时发现，默认计划为每周一 03:15，也支持超级管理员在页面点击“立即发现”手动触发。

## 3. 候选数据集生成

每个公开 API 返回的结果会被统一转换为候选数据集记录。系统会尽量抽取以下信息：

| 字段 | 说明 |
|---|---|
| title | 数据集标题 |
| doi | 数据集 DOI 或论文 DOI |
| paperUrl | DOI 页面或论文页面 |
| dataUrl | 文件下载链接或数据入口 |
| repository | 来源仓库，如 Zenodo、Figshare、NOMAD |
| license | 许可证信息 |
| dataFormat | 通过文件名、链接或元数据推断的数据格式 |
| dataScale | 文件大小或待确认规模 |
| method | 推断的计算方法，如 DFT、VASP、ORCA、ML potential 等 |
| detectedFields | 推断出的字段，如 structure、energy、force、bandgap |
| parserPlan | 建议的解析方案 |

不同来源的元数据结构不同，因此后端先把它们归一化为统一的 `DiscoveryCandidateResponse`，再进入评分和审核流程。

## 4. 自动评分与自动校对

候选数据集会经过两类自动判断：综合评分和校对报告。

### 4.1 综合评分

评分主要考虑：

- 是否有 DOI；
- 是否有数据下载链接；
- 是否有许可证；
- 文件格式是否可识别；
- 是否包含结构、能量、力、带隙等计算材料字段；
- 是否和材料计算、DFT、VASP、高分子数据库主题相关；
- 是否可能重复收录。

评分高的数据集会排在候选列表前面。若自动转入规则开启，高分候选可自动转入接入审核队列，但不会自动发布。

### 4.2 自动校对

自动校对包括：

| 校对项 | 当前判断方式 |
|---|---|
| 数据链接 | 判断是否存在 HTTP/HTTPS 数据入口 |
| 许可证 | 判断是否包含 CC BY、CC0、MIT、Apache 等常见开放许可证关键词 |
| DOI | 判断 DOI 是否规范 |
| 格式 | 判断是否识别出 HDF5、CSV、JSON、CIF、XYZ、POSCAR 等格式 |
| 物理字段 | 判断元数据中是否出现 structure、coordinates、energy、force、stress、gap 等关键词 |
| 相关度 | 根据标题、描述、字段和方法判断是否属于计算材料领域 |
| 重复性 | 用 DOI 查询当前展示库中是否已有相关记录 |

校对结果分为：

- `PASS`：初步通过；
- `WARN`：存在提醒，需要人工确认；
- `FAIL`：关键项不通过，通常阻止自动转入队列。

## 5. 超级管理员转入接入队列

候选数据集不会直接进入正式数据库。超级管理员需要在“数据接入中心”查看候选详情，决定是否转入接入队列。

转入队列后，系统会创建一条数据集投稿记录，来源为 `auto-discovery`。这一步只代表“值得进一步审核”，不代表已经发布。

## 6. 来源审核

进入接入队列后，超级管理员需要审核来源。

审核重点：

- 论文或 DOI 是否真实；
- 数据链接是否可访问；
- 数据和论文是否对应；
- 许可证是否允许公开展示和下载；
- 是否与当前数据库主题相关；
- 是否已被当前平台收录；
- 是否有必要开发专门解析器。

审核结果：

- `APPROVED`：来源通过，进入后续适配流程；
- `REJECTED`：来源退回，不进入接入流程。

## 7. 读取样本并格式预检

来源通过后，管理员可以触发“读取样本并解析预检”。

这里的“读取样本”不是把完整数据集保存到本地，而是从数据下载链接读取受控大小的数据片段到内存中，用于快速判断格式、字段和结构可解析性。预检结果会写入接入库，原始文件默认不落盘。

当前默认配置：

- 小文件：最多读取 `VASP_DISCOVERY_AUTOADAPT_MAX_MB`，默认 50 MB；
- 大文件：使用 HTTP Range 只读取前 `VASP_DISCOVERY_AUTOADAPT_SAMPLE_MB`，默认 4 MB；
- 读取内容只用于预检，不等同于完整下载、完整解压或正式入库；
- 如需保存完整原始文件，应增加独立的“受控下载与校验”步骤，记录本地路径、文件大小、hash 和解压状态。

当前支持的预检格式：

| 格式 | 当前预检能力 |
|---|---|
| CSV / TSV | 读取表头，识别列名和记录数估计 |
| JSON | 读取 JSON 对象或数组，识别字段 |
| JSONL / NDJSON | 按行读取 JSON 对象，识别首条字段和记录数估计 |
| XYZ / EXTXYZ | 读取首帧原子数、元素、坐标；EXTXYZ 可识别 energy、forces、stress、lattice |
| CIF | 读取化学式、元素、晶胞、分数坐标并可转笛卡尔坐标 |
| POSCAR | 读取元素、原子数、结构基本信息 |
| HDF5 | 读取 group/dataset 结构；完整文件可进一步解析 coordinates、species、energies、forces、lattice |
| ZIP | 读取压缩包内部文件名，并优先选择内部 CIF/XYZ/HDF5/JSON/CSV 等文件继续预检 |
| TAR | 读取归档内部文件名，并优先选择可解析文件继续预检 |
| GZIP / XZ / BZIP2 | 解压一层样本；若内部是 TAR，会继续进入 TAR；若内部是表格或结构文件，会继续按内部格式预检 |
| ZSTD | 当前只识别外层格式，后续可增加 Zstandard 解压依赖后继续解析 |

大文件策略：

- 若文件大小在配置上限内，系统尝试完整下载；
- 若文件过大，系统只下载 Range 样本；
- HDF5 如果被 Range 截断，不会强行解析结构，会提示提高下载上限或人工下载完整文件。

## 8. 字段映射或结构解析

预检完成后，系统会生成入库建议。

### 8.1 表格型数据

适用于 CSV / TSV / JSON / JSONL。

管理员需要确认字段映射，例如：

| 展示库字段 | 来源字段示例 |
|---|---|
| MATERIAL_ID | id、material_id、mpid |
| MATERIAL_NAME | name、title |
| COMPOSITION | formula、composition |
| ATOM_COUNT | natoms、atom_count |
| ENERGY | energy、total_energy |
| HOMO / LUMO / GAP | homo、lumo、gap |
| DOI | doi |

字段映射确认后，系统可写入前若干条记录。

### 8.2 结构型数据

适用于 XYZ、EXTXYZ、CIF、POSCAR、HDF5。

这些格式可由结构适配器自动生成：

- `STRUCTURE_JSON`
- `ATOM_COUNT`
- `COMPOSITION`
- `ENERGY`，如来源提供
- `FORCES_JSON`，如来源提供
- `lattice`，如来源提供

不同格式能力如下：

| 格式 | 当前结构解析能力 |
|---|---|
| XYZ | 元素和坐标 |
| EXTXYZ | 元素、坐标、energy、forces、stress、lattice |
| CIF | 元素、分数坐标转笛卡尔坐标、晶胞盒 |
| POSCAR | 元素和原子数，结构信息仍较基础 |
| HDF5 | 支持 coordinates / positions、species / atomic_numbers、energies、forces、lattice |

## 9. 半自动写入展示库

管理员确认后，系统将解析出的记录写入统一展示库。

写入规则：

- 数据集 ID 使用 `intake_投稿编号`；
- 默认最多写入前若干条记录，避免误写入大规模异常数据；
- 写入前会删除同一 `intake_编号` 的旧记录，实现覆盖式重入库；
- 写入目标为 H2 展示库 `display_records`；
- 写入后可在数据中心看到该新数据集。

注意：这一步仍属于半自动接入，不等同于最终质量认证。

## 10. 质量验证

入库后，数据集会进入质量验证页面。

质量验证会从展示库统计：

- 标识字段覆盖率；
- 组成字段覆盖率；
- 原子数字段覆盖率；
- 三维结构覆盖率；
- 能量字段覆盖率；
- 力字段覆盖率；
- DOI / 数据链接覆盖率；
- 软件、泛函、基组等计算证据覆盖率；
- 结构抽样合法性；
- 疑似重复签名；
- 单位和方法证据完整度；
- 解析适配成熟度。

系统会生成：

- 综合质量分；
- 风险等级；
- 发布建议；
- 缺失字段；
- 待处理质量问题；
- 数据集质量报告。

## 11. 发布、隐藏与撤回

超级管理员可以对数据集做最终治理操作：

| 操作 | 含义 |
|---|---|
| 发布 | 普通用户可在数据中心看到该数据集 |
| 隐藏 | 数据仍在库中，但普通用户不可见 |
| 撤回入库 | 删除 `intake_编号` 对应展示记录，回到可重新入库状态 |

发布前应确认：

- 来源可信；
- 许可证可接受；
- 记录解析正确；
- 字段和单位清晰；
- 质量分和风险项符合展示要求；
- 必要时已补充人工说明。

## 12. 角色权限

| 角色 | 权限 |
|---|---|
| 游客 | 查看公开页面、了解接入流程 |
| 注册用户 | 提交数据集线索，查看自己的投稿进度 |
| 管理员 | 查看候选发现和质量审核相关信息 |
| 超级管理员 | 运行自动发现、修改发现配置、审核来源、转入队列、入库、撤回、发布/隐藏 |

## 13. 当前已经完成的能力

当前系统已经具备：

- 多公开 API 数据集发现；
- 候选数据集统一建模；
- 自动评分；
- 自动校对；
- 候选转入接入队列；
- 注册用户投稿；
- 超级管理员来源审核；
- 样本下载和格式预检；
- CSV / JSON 字段映射入库；
- XYZ / EXTXYZ / CIF / POSCAR / HDF5 结构解析入库；
- 展示库覆盖式写入；
- 入库撤回；
- 数据质量验证；
- 发布 / 隐藏控制。

## 14. 当前仍未完成的闭环

目前系统尚未做到完全自动化。仍未完成或需要加强的部分包括：

- 自动下载完整大数据集；
- 自动解析所有复杂 HDF5 / LMDB / AiiDA / NOMAD archive 语义；
- 自动单位识别与单位换算；
- 自动调用 ASE / pymatgen 做全量结构合法性验证；
- 自动结构去重和结构哈希；
- 自动异常键长、异常能量、异常元素组合检测；
- 自动重建完整 H2 展示库；
- 自动上线发布；
- 完整审计日志和版本回滚；
- 针对 Materials Project、AFLOW、OQMD、Materials Cloud 的专用深度连接器。

因此，当前定位是：

> 自动发现和半自动接入系统，而不是全自动无人值守数据发布系统。

## 15. 推荐的专家审核原则

进入正式发布前，建议超级管理员至少确认：

1. 数据来源和 DOI 可追溯；
2. 数据许可证允许平台展示和下载；
3. 数据集与 VASP / DFT / 计算材料 / 高分子 AI 主题相关；
4. 解析字段与原始数据语义一致；
5. 能量、力、坐标、晶胞、带隙等单位明确；
6. 抽样结构能正确显示；
7. 质量验证无阻断项；
8. 数据集介绍、论文链接、数据链接完整；
9. 如只接入样本或索引，应在页面明确说明；
10. 发布后保留撤回和重入库路径。
